# 후쿠부쿠로 (Fukubukuro) 프로젝트

## 📋 프로젝트 개요

**가상 피팅룸 기반 온라인 쇼핑 플랫폼**

사용자가 자신의 체형 정보(신장/치수)를 입력하면 AI 기반 가상 피팅을 통해 옷이 어떻게 보일지 미리 확인할 수 있는 데스크톱 쇼핑 서비스

---

## 🔄 전체 프로젝트 흐름

### 사용자 흐름 (서비스 관점)

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 1. 체형입력  │───▶│ 2. 상품검색  │───▶│ 3. 상품상세  │
│ (신장/치수)  │    │(키워드/필터) │    │(이미지/정보) │
└─────────────┘    └─────────────┘    └─────────────┘
                                            │
                                            ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 6. 추천/랭킹 │◀───│ 5. 결과보기  │◀───│ 4. 피팅요청  │
│(조회수/좋아요)│    │ (전/후 비교) │    │(AI서버 호출) │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 개발 흐름 (진행 관점)

```
기획/설계 → 코어개발 → AI프로토타입 → 연동 → 고도화 → 테스트/발표
(요구사항   (쇼핑몰     (try-on API)  (E2E    (추천/   (시연/
 UI/ERD     기본+검색)                 1번     랭킹/    영상)
 API)                                 흐름)   UX)
```

> **핵심 원칙**: "매주 '사용자 흐름' 1개씩 완성도를 올린다"

## 👥 팀 역할 분담 (5인)

| 역할 | 주담당 (Owner) | 개발 필수 | 부담당 (Backup) |
|------|---------------|----------|----------------|
| **팀장** | PM/아키텍처, API설계, 코드리뷰, 통합관리 | 회원/인증, 상품/주문 핵심 모듈 1~2개 | B의 AI 연동 마무리 |
| **A** (프론트) | 화면설계, 디자인시스템, 프론트 구현 | API 연결 페이지 2개 이상 | D와 페어로 UI 개발 |
| **B** (AI/연동) | Python AI 서버, 프롬프트 설계, 입출력 규격 | AI 서버 endpoint + Spring 연동 | C의 추천/랭킹 데이터 규격 |
| **C** (DB/백엔드) | ERD/테이블명세, 검색/랭킹/추천 구현 | Repository/Service 레벨 기능 | 팀장 코어 모듈 보조 |
| **D** (QA/영상) | 테스트 시나리오, 시연 흐름, 영상 편집 | 작은 기능 1개 (UI 컴포넌트 또는 간단한 API) | A와 페어로 UI 작업 |

## 📦 기능 세트별 담당

| 기능 세트 | Owner | Backup |
|----------|-------|--------|
| **쇼핑몰 코어** (상품목록/상세/장바구니/주문) | 팀장 | C |
| **검색/최근검색/추천** | C | D(프론트UI) + B(데이터규격) |
| **가상 피팅** (신장/치수 입력 → 피팅 요청 → 결과 표시) | B | 팀장(연동) + A(피팅UI) |
| **UI/UX** (피팅룸UI, 로딩/에러, 전/후 비교) | A | D |
| **발표/시연/영상** (일본어 자막, BGM, 5분 시나리오) | D | 전원 |

## 📅 주간 목표 (6주)

| 주차 | 목표 | 완료 기준 |
|------|------|----------|
| **1주차** | 기획/설계 완료 + 기술 스파이크 | 요구사항, UI, ERD, API 확정 + 연동 시범 |
| **2주차** | 쇼핑몰 코어 | 상품/검색/상세 화면·API 연결 |
| **3주차** | AI 서버 프로토타입 + 피팅 화면 | **E2E 1번 흐름 완성** (더미여도 OK) |
| **4주차** | 추천/랭킹/UX | 최근검색/추천/랭킹 중 2개 + UX 다듬기 |
| **5주차** | 통합테스트 + 발표 준비 | 버그픽스 + 시나리오/영상 초안 |
| **마감 1주 전** | Code Freeze | 기능 동결 → 발표/영상/문서 마무리 |

## 📋 일일 운영 규칙

### 규칙 A: 매일 카드 1장만 확실히 끝낸다
- 각자 오늘의 카드 1개(Owner) + 백업 1개(Backup)만 잡기
- 카드에는 **완료조건(Definition of Done)** 필수

### 규칙 B: 하루 2번만 공유한다 (짧게)
- **오전**: "오늘 할 것" 3줄
- **종료 전**: "완료/미완료/막힌 점" 3줄

### 규칙 C: 카드 완료조건(DoD) 통일
카드가 "완료"가 되려면 최소:
- [ ] 실행 화면 or API 동작 확인 (스크린샷/짧은 영상도 OK)
- [ ] 에러/예외 처리 1개 이상
- [ ] 관련 문서 1줄 업데이트 (요구사항/화면설계/API명세 중 하나)

### 규칙 D: 초보자는 항상 페어가 있다
- 초보자는 "문서/영상만" 금지
- 반드시 작은 기능 1개 오너 + A 또는 C와 페어 진행

### 규칙 E: "막히면 30분 룰"
- 30분 막히면 바로 공유 → 팀장이 조정 (대기시간 방지)

## 👤 역할별 매일 할 일

| 역할 | 매일 할 일 |
|------|-----------|
| **팀장** | "오늘의 우선순위 3개" 공지 + 막힌 카드 해결 + 통합/리뷰 1건 |
| **A** (프론트) | 화면 1개 (검색/상세/피팅/추천 중) 완성 + API 연결 확인 |
| **B** (AI/연동) | AI API 1개 개선 (입력값/응답/에러) 또는 연동 1개 진전 |
| **C** (DB/백) | 테이블/쿼리/집계 (최근검색/랭킹/추천) 중 1개 끝내기 |
| **D** (QA/영상) | 테스트 시나리오 2개 추가 + 버그 1개 재현/정리 + 작은 UI/기능 1개 (페어) |
